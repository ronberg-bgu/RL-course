Question 1: 
How does the `(push-big)` strict action constraint enforce that exactly two adjacent agents push the large box? What happens if three agents try to push it in the RL environment vs the PDDL domain?**

Answer 1:

Each push-big-{up,down,left,right} action takes exactly two agent parameters (?a1 ?a2 - agent) and enforces:

Each agent must be directly behind their respective box tile:
(agent-at ?a1 ?from1) (adj-up ?from1 ?boxloc1)
(agent-at ?a2 ?from2) (adj-up ?from2 ?boxloc2)

The big box spans exactly two horizontally adjacent cells:
(bigbox-at ?b ?boxloc1 ?boxloc2) (adj-right ?boxloc1 ?boxloc2)
There are only two box tiles, one per agent parameter slot.

Both destination cells must be clear simultaneously, enforcing coordination.

A third agent cannot be added - the schema is fixed-arity with no third agent slot.

What Happens with a Third Agent?

PDDL domain: The third agent is simply ignored. No push-big variant with 3 agent parameters exists, so the planner will never generate such an action. The third agent can only move out of the way.

RL environment: In the RL environment (multi_agent_env.py), pushing a heavy box requires `len(pushers) >= 2`, all pushers sharing the same direction `(len(dirs) == 1)`, and all originating from the same grid cell `(len(origins) == 1)`. If three agents all satisfy these conditions simultaneously, the RL environment accepts it - the box moves and all three agents are placed at the box's former position. So a third agent doesn't cause chaos; it participates in the push. If the conditions aren't met (e.g., different directions or origins), the push simply fails and no movement occurs.

Key Takeaway: PDDL enforces the 2-agent constraint by construction through fixed-arity parameters and geometry predicates. The RL environment does not - a third agent is an emergent physics situation with unpredictable results.
